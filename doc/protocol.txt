Recieveable by server and client:

{"type": "PING"}:
Sends PONG back.

{"type": "PONG"}:
Outputs a connection confirmation.

{"type": "ERROR", "msg": str}:
Indicates that the previous inquiry caused an error. With "msg" being a more
detailed message.

{"type": "CLOSE"}:
Indicates that the other side will close the connection.



Recieveable by server only:

{"type": "NAME", "name": str}:
Sets the name of the player sending the package to "name".

{"type": "INFO"}:
Replies with general information like list of player names.

{"type": "INFO_DETAIL", "src": str|int}:
Replies with the list of cards the source has. The source can be the name or index of a
player or "BANK". The server will attempt to interpret str as an int.

{"type": "BUY", "card": str|int}:                               WIP
Buys the card. The source can be the name or index of a card. The server will attempt to
interpret str as an int.



Recieveable by client only:

{"type": "PHASE", "phase": int}:
Notifies client of being active and of the current phase. Server expects an appropriate answer to
this.

{"type": "PRINT", "msg": str}:
Outputs "msg".

{"type": "PRINTS", "lines": [(str_id, str_format, args..., )]}:
Outputs each tupel as a formated line.



Console input:

name str
Send a NAME package to the server after some checks.

info
Requests info.

info_detailed str|int
Requests detailed info for the source.

buy str|int
Attempts to buy the card.
